Text Fields

Text Fields

- a) input elements-> be used to accept data from the user
- has different types -> <input type="..." />
- - text
- - number -> only accept numbers
- - password
- - date
- - email -> only accept emails

* checkbox | radio | range | file | hidden -> other types of input element

- b) textarea element -> to type multiple lines of text -> <textarea placeholder="..."></textarea>
- - it has different attributes -> - name (send to server), - id (for lable), - cols, - rows
- - * stop resizing textarea box -> css -> textarea { resize: none; }

different important atteibutes of input & textarea fields
- value -> to fill an input field automatically -> e.g. email
- - for textarea we should put value in between the tags -> <textarea>...</textarea>
- placeholder -> e.g. comment -> it can be used insted of label
- read only -> can not be change but will be sent to server
- disabled -> will not be sent to server
- maxlength -> e.g. password
- autofocus -> The first field gets focus automatically -> e.g. name

Data Lists -> suggestion

1. add a text field -> <input type="text" />

2. add a datalist element -> <datalist> ... <datalist />

3. inside of datalist element add some options -> <option> ... <option /> -> e.g. name of countries

4. link data list to the input field.
- - a) give the data list an id -> <datalist id="...">
- - add list attribute to input element -> <input type="text" list="..." />

5. data-value -> send the value of selected part to the server -> <option data-value="...">

Drop-down Lists

1. add select element to the form -> <select> ... <select />

2. inside of select element add some options -> <option> ... <option />

3. to make the drop-down list optional -> leave the value of first option empty

- - * to be seleted an option bei default -> add selected attribute to the option -> <option value=" " selected>...</option>

- - * allow to select multiple options -> add multiple attribute to select element -> <select multiple>...</select>

- - * to categorize the options -> use optgroup element -> <optgroup>...</optgroup>
- - * each optgroup should have a label -> <optgroup label="...">

Check Boxes : a type of input

1. add input element -> <input type="..." id="..." />

2. set the type attributet to checkbox -> <input type="checkbox" id="..." />

3. need a lable element right after that -> <label for="...">...</label>

4. link label with input -> id (input) = for (label)

5. add class="label-inline" to lable element -> label is bei default a block level element

Sliders (range selectors)

1. add input element -> <input type="..." />

2. set the type attributet to range -> <input type="range" min="..." max="..." />

3. set min attributet to 0 and max to 100 -> <input type="range" min="0" max="100" />

File Inputs

* multiple attribute -> the user can select multiple files -> <input type="file" multiple />
* accept attribute -> which keind of files -> <input type="file" accept=".jpg, .png" />

Data Validation

1. befor submiting a form, we should always validate the data

2. html5 elements com with built in validation -> e.g. <input type="text" required /> -> required

examples of validation for basic text fields:
- required
- minlength
- maxlength -> should always use it

examples of validation for some other fields:
- type="email"
- type="date"

for number we have two atteibutes -> <input type="number" min="0" max="..." />
- min -> should always use it, especially if you don't allow negative values
- max

Submitting the Form

for submitting a form:

1. 2 atteibutes should be set of the form element -> <form action="..." method="..." ></ form>

action -> the address where data is send (server)
- create a backend for the form
- - - go to formspree
- - - add a new project
- - - add a new form
- copy the link of new form
- paste the URL to action attribute

method -> how the data is sent
- - POST -> form data will be include in the body of the HTTP request -> the most common approach
- - GET -> form data is included in the URL -> if we want to bookmark the URL

2. each input field should have a name attribute -> <input name="..." />
